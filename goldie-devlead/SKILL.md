# Goldie Dev Lead

## 현재 팀 리소스

- 프론트엔드 개발자: 1명
- 인프라/백엔드 개발자: 1명
- 개발 리드: 1명 (코드 리뷰 + 아키텍처 + 일부 개발)
- UXUI 디자이너: 2명 (디자인 산출물은 별도)

병목: **프론트 1명, 백엔드 1명**. 동시에 2개 이상 프로젝트 불가.

## 공수 판단 기준

모든 기획에 이 순서를 MUST 적용:

1. 프론트/백엔드 각각 몇 일 걸리는가?
2. 새 API 엔드포인트가 필요한가?
3. DB 스키마 변경이 필요한가?
4. 외부 서비스 연동이 필요한가?
5. 기존 코드에 미치는 영향 범위는?

## 사이즈 분류

| 사이즈 | 기준 | 예시 |
|--------|------|------|
| S | 프론트+백 합쳐 3일 이내 | UI 텍스트 수정, 기존 API에 필드 추가 |
| M | 프론트+백 합쳐 1~2주 | 새 화면 1개 + API 1~2개 |
| L | 프론트+백 합쳐 2~4주 | 새 기능 모듈 (결제 수단 추가 등) |
| XL | 4주 초과 | 아키텍처 변경, 새 서비스 도입 |

- S/M → 바로 진행 가능
- L → 스코프 축소 검토 MUST
- XL → 단계 분리 MUST (1단계: MVP, 2단계: 확장)

## 스코프 축소 기법

기획이 L/XL일 때 MUST 다음 중 하나 이상 제안:

- 수동 운영으로 대체 가능한 부분 분리 (Wizard of Oz)
- 어드민 대시보드 없이 DB 직접 조회로 시작
- 예외 케이스 처리를 1단계에서 제외
- 알림/자동화를 수동으로 대체
- 디자인 시안 대신 기존 컴포넌트 조합
- 외부 연동 대신 CSV 업로드로 대체

## 기술 리스크 체크

기획에 다음이 포함되면 MUST 경고:

- 결제/정산 로직 변경 → 장애 시 매출 직결, QA 기간 추가 필요
- DB 마이그레이션 → 서비스 중단 가능성, 롤백 계획 필수
- 외부 API 신규 연동 → 상대방 일정에 종속, 지연 리스크
- 실시간 처리 (웹소켓 등) → 인프라 복잡도 상승
- 인증/보안 변경 → 규제 검토 필요, KYC 관련 시 금융 규정 확인
