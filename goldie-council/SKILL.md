# Goldie Council

## 현재 상황

- 초기 서비스, 검증 단계, 소규모 모수
- 팀 7명: UXUI 2 / 프론트 1 / 백엔드 1 / 개발리드 1 / PM 1 / DA겸마케터 1
- 모든 결정에 "이 팀으로 실행 가능한가?"를 MUST 고려
- 대규모 실험/프로젝트는 현실적이지 않음

## 사전 준비

회의 시작 전 MUST 네 스킬 파일을 읽는다:
- `~/.claude/skills/goldie-pm/SKILL.md`
- `~/.claude/skills/goldie-po/SKILL.md`
- `~/.claude/skills/goldie-da/SKILL.md`
- `~/.claude/skills/goldie-devlead/SKILL.md`

## 회의 절차

### 라운드 1 — 입장 표명

각 역할이 자기 스킬의 출력 형식대로 독립적으로 의견을 낸다.
다른 역할의 의견을 참고하지 않는다.

```
🟦 PM 의견
(goldie-pm 출력 형식)

🟩 PO 의견
(goldie-po 출력 형식)

🟧 DA 의견
(goldie-da 출력 형식)

🟥 Dev Lead 의견
(goldie-devlead 출력 형식)
```

### 라운드 2 — 반박

각 역할이 다른 두 역할에게 근거 기반으로 반박 또는 질문한다.

```
🟦 PM → 다른 역할: [반박/질문]
🟩 PO → 다른 역할: [반박/질문]
🟧 DA → 다른 역할: [반박/질문]
🟥 Dev Lead → 다른 역할: [반박/질문]
```

각 역할은 가장 이견이 큰 1~2개 역할에 집중 반박한다.

### 라운드 3 — 수정 입장

반박을 반영해 각 역할이 입장을 수정하거나 유지한다.

```
🟦 PM 수정 입장: [유지/수정 + 이유]
🟩 PO 수정 입장: [유지/수정 + 이유]
🟧 DA 수정 입장: [유지/수정 + 이유]
🟥 Dev Lead 수정 입장: [유지/수정 + 이유]
```

### 최종 결론

```
📋 최종 결론
━━━━━━━━━━━━━━━━━━━━━━━
합의 사항: [동의하는 부분]
미합의 사항: [의견 갈리는 부분 + 각 입장]
결정: [최종 추천 행동]
근거: [왜 이 결정인지]
실행 계획: [누가 / 며칠 / 어떻게]
다음 단계: [구체적 액션 아이템]
━━━━━━━━━━━━━━━━━━━━━━━
```

## 규칙

- 네 역할 모두 찬성 → MUST 놓친 관점 없는지 재확인
- DA가 "데이터 부족" → 소규모에서 가능한 검증 방법 먼저 설계
- Dev Lead가 "L/XL" 판정 → 스코프 축소 논의 MUST 선행
- 결론 불가 → "보류 + 필요 검증 사항"도 유효한 결론
- **실행 계획 없는 결론 금지** — 반드시 담당자와 일정 명시
- **Dev Lead가 불가 판정한 스코프를 그대로 결론 내는 것 금지**

## 검증 체크리스트

회의 완료 후 MUST 확인:
- [ ] 3라운드(입장→반박→수정) 모두 진행했는가?
- [ ] 각 역할이 자기 스킬의 출력 형식을 따랐는가?
- [ ] 반박이 감정이 아닌 근거 기반인가?
- [ ] 최종 결론에 실행 계획(누가/며칠)이 포함되었는가?
- [ ] 7명 팀으로 2주 내 실행 가능한 결론인가?
- [ ] 검증 방법이 소규모 모수에 맞는가?

하나라도 빠지면 → 해당 라운드 보완 후 재출력
